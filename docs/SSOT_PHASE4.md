# SSOT Phase4 (v0.1)

## Phase4 の目的
- Phase3 の設計成果物を「実装」する

## Phase4 でやること（実装対象）
- Phase3 SSOT に記載された設計成果物を実装する

## Phase4 でやらないこと
- Phase0/1/2/3 の仕様変更
- 未決定事項の推測による実装
- Phase3 設計の再解釈

## 人間判断の確定事項（Phase4 START判定用）
- 運用フローの定例頻度: 週次（緊急時のみ随時）
- 承認基準: 送らないをデフォルトとし、明示条件をすべて満たす場合のみ送信
- Phase4 実装境界: Phase3 SSOT に記載された設計成果物の忠実な実装のみ

## Phase3 からの持ち越し事項（確定済み）
- 人間運用フローの定例頻度（日次/週次など） → 週次（緊急時のみ随時）
- 承認基準（送信可否の判断条件） → 送らないデフォルト／条件満たす場合のみ送信
- Phase4 以降の実装方針と境界 → Phase3 SSOT 設計成果物の忠実実装のみ

## Phase4 開始前に人間が決める必要がある事項（チェックリスト）

- [x] Phase4 実装の境界（Phase3 設計成果物の忠実実装のみ）

## 前提（参照のみ）
- Phase3 は CLOSE（設計完了・実装禁止）
- Phase0/1/2 は immutable
