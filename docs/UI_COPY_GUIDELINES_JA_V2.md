# UI_COPY_GUIDELINES_JA_V2

Journey/Task系のユーザー表示文言ガイド（add-only）。

## Rule 1: 内部語を表示しない
- 禁止: `DUE_SOON`, `due_soon_tasks`, `next_tasks` など内部識別子
- 表示語: `今週の期限`, `期限（7日以内）`, `期限超過`, `今日の3つ`

## Rule 2: 行動不能理由を明示する
- `blockedReason` を持つタスクは `ブロッカー:<reason>` を表示する
- ユーザーが次の一手を選べる状態を優先する

## Rule 3: 相談導線の意味を固定する
- `相談` は案内表示と利用イベント記録のみ
- コマンド実行時点でチケット作成しないことを明記する

## Rule 4: 期限表示は2面で返す
- 期限の近いもの（7日以内）
- 期限超過

## Canonical examples
- `期限のある未完了タスク:`
- `【期限（7日以内）】`
- `【期限超過】`
- `カテゴリ別TODO件数:`
- `相談導線を開きました。`
